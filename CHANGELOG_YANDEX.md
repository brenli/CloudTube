## Как это работает

### Процесс подключения

1. **Пользователь выполняет команду:**
   ```
   /connect https://webdav.yandex.ru email@yandex.ru oauth_token
   ```

2. **Бот сохраняет учетные данные:**
   - В `~/.davfs2/secrets` с правами 600
   - В базу данных SQLite

3. **Бот монтирует диск:**
   ```bash
   sudo mount -t davfs https://webdav.yandex.ru /mnt/yandex-disk -o uid=USER,gid=USER
   ```

4. **Проверка подключения:**
   - Попытка чтения содержимого `/mnt/yandex-disk`
   - Получение информации о свободном месте

### Процесс загрузки

1. **Пользователь отправляет ссылку на видео**

2. **Бот скачивает видео:**
   - Во временную директорию (по умолчанию `/tmp`)
   - С отслеживанием прогресса (0-90%)

3. **Бот загружает на Яндекс.Диск:**
   - Копирование в `/mnt/yandex-disk/Single Videos/` или `/mnt/yandex-disk/Playlist Name/`
   - С отслеживанием прогресса (90-100%)
   - Чанками по 1MB для больших файлов

4. **Очистка:**
   - Удаление временного файла
   - Обновление статуса задачи

## Устранение проблем

### Проблема: "mount.davfs: mounting failed"

**Причины:**
1. Неверный OAuth токен
2. Токен истек (срок действия 1 год)
3. Нет доступа к интернету

**Решение:**
```bash
# Получите новый токен
python get_yandex_token.py

# Обновите учетные данные
nano ~/.davfs2/secrets

# Попробуйте снова
/connect https://webdav.yandex.ru email@yandex.ru new_token
```

### Проблема: "Permission denied"

**Причины:**
1. Неправильные права на точку монтирования
2. Не настроен sudo

**Решение:**
```bash
# Исправьте права
sudo chown $USER:$USER /mnt/yandex-disk

# Настройте sudo (см. YANDEX_DISK_SETUP.md)
sudo visudo
```

### Проблема: Медленная загрузка

**Причины:**
1. Медленное интернет-соединение
2. Маленький cache_size в davfs2

**Решение:**
```bash
# Увеличьте cache_size
nano ~/.davfs2/davfs2.conf

# Измените:
cache_size 200  # вместо 50
```
